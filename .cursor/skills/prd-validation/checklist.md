# PRD Validation Checklist

Use when performing gap analysis. Tick or note gaps per area.

## Product fundamentals
- [ ] Vision and problem statement clear
- [ ] Target users and primary needs defined
- [ ] Success metrics stated and measurable
- [ ] Scope (in/out) and boundaries explicit

## Technical requirements
- [ ] Tech stack or constraints stated
- [ ] Integrations and APIs called out
- [ ] Security, performance, scalability addressed
- [ ] Infrastructure or deployment noted

## Business considerations
- [ ] Timeline or milestones present
- [ ] Budget or resource constraints mentioned
- [ ] Regulatory/compliance needs (if any)
- [ ] Market or business model context (if relevant)

## Implementation factors
- [ ] Dependencies and third-party requirements listed
- [ ] Team/skills assumptions stated
- [ ] Testing approach indicated
- [ ] Deployment/release approach indicated

## Structure (align with prd-creation)
- [ ] Overview, Goals, Scope, Personas
- [ ] Functional requirements (with priority)
- [ ] Non-functional requirements
- [ ] User journeys, Success metrics, Timeline
- [ ] Open questions / assumptions
