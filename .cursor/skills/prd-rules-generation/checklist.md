# RULES.md Generation Checklist

Use before finishing RULES.md generation.

## Pre-Generation

- [ ] PRD and features list have been read
- [ ] Critical missing information has been identified; questions asked if needed
- [ ] Project overview drafted and accurate

## Content Completeness

- [ ] **Technology stack** — All core technologies and versions specified
- [ ] **Naming** — Conventions for files, components, variables, types
- [ ] **Structure** — Folder layout, modularity, separation of concerns
- [ ] **Architecture** — Patterns, data flow, boundaries
- [ ] **Data/API** — Handling, validation, state, error representation
- [ ] **Performance** — Budgets, benchmarks, optimization expectations
- [ ] **Security** — Input validation, secrets, auth (if applicable)
- [ ] **Testing** — Types, coverage, mocking
- [ ] **Documentation** — Inline, API docs, README
- [ ] **Errors/Logging** — Patterns, levels, format
- [ ] **Accessibility** — If UI is in scope
- [ ] **Responsive design** — If UI is in scope
- [ ] **Priorities** — Core vs enhancements, phases, quality thresholds
- [ ] **General** — Requirements adherence, quality, completeness, uncertainty handling

## Quality

- [ ] Rules are specific and actionable
- [ ] No vague or contradictory guidance
- [ ] Examples included where they add clarity
- [ ] PRD constraints reflected in rules
- [ ] Format is clean and logically organized
