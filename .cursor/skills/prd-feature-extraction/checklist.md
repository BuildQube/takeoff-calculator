# Feature Extraction Checklist

Use this for a systematic pass when extracting features from a PRD.

## Before extraction
- [ ] PRD read in full
- [ ] Product overview drafted
- [ ] Critical gaps identified; questions asked if needed

## Per feature
- [ ] Unique ID assigned (F1, F2, …)
- [ ] Single, discrete scope
- [ ] MoSCoW priority set
- [ ] Category and persona (if any) set
- [ ] Description clear for technical and non-technical readers
- [ ] Acceptance criteria testable
- [ ] Technical notes and edge cases noted
- [ ] Complexity (Low/Medium/High) and integrations/risks noted

## Document
- [ ] Table of contents with working links
- [ ] Summary counts by priority and category correct
- [ ] All feature IDs listed in “By category” and “By priority”
- [ ] features.md saved at agreed path (default: workspace root)
